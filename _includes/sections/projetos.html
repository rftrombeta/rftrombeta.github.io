🔗 Estratégia: Co-location (Full-Stack Testing)

Arquitetura co-location full-stack em repositório único, com suíte Playwright acoplada ao ciclo de desenvolvimento da aplicação para feedback rápido em alterações de front e back-end.

Implementação conduzida durante o treinamento AutomatizAi (Fernando Papito), aplicando de forma incremental as práticas de estruturação, automação e validação contínua demonstradas no programa.

🏗️ Estratégia: Shared Libraries (Scalable Ecosystem)

Shared Library Python centralizada para contratos e geração de dados de teste, com tipagem via Pydantic e estratégias determinísticas de data factory com Faker.

Core Lib Python Pydantic

Projeto consumidor em Robot Framework que versiona a Core Lib Python via Git, isolando regras de domínio da camada de execução e promovendo reúso entre suítes. Execução contínua em CI com relatórios consolidados para rastreabilidade de qualidade.

Robot Framework Consumer CI Pipeline

Biblioteca Core em TypeScript publicada como pacote NPM, com utilitários, tipagens e integrações para automação em ecossistema Node.js. Estrutura orientada a reúso entre projetos e compatível com Playwright como runner principal.

Core Lib TypeScript NPM Package

Projeto consumidor em evolução para validação da Core Lib TypeScript em cenários de integração ponta a ponta, cobrindo fluxo de uso real e estabilidade dos contratos públicos.

🧩 Estratégia: Framework Modular (POM)

Framework modular baseado em Page Object Model para testes de API com Playwright e TypeScript, priorizando separação de camadas e manutenção de longo prazo. Pipeline CI acionada por commit com histórico de execuções e artefatos de reporte.

Playwright TypeScript POM CI Pipeline

Framework E2E com Playwright + Cucumber, modelado em Gherkin e estruturado por módulos para suportar escalabilidade de cenários e legibilidade funcional. Execução padronizada em CI para validação regressiva e visibilidade contínua dos resultados.
